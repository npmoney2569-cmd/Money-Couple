# LINE Bot Integration

> Phase 3 — หลัง Auth และ LINE Login พร้อมใช้งาน

## แนวคิด

ผู้ใช้พิมพ์ข้อความหรือส่งรูปใบเสร็จเข้าแชท LINE Official Account ของระบบ บอทแยกวิเคราะห์และบันทึกรายรับ-รายจ่ายอัตโนมัติ พร้อมตอบกลับยืนยันในแชท

## Flow การทำงาน

1. ผู้ใช้พิมพ์ข้อความในแชท LINE เช่น "ข้าวมันไก่ 45" หรือส่งรูปใบเสร็จ
2. LINE ส่ง event มาที่ Webhook ของระบบ
3. Backend รับข้อความ ตรวจสอบ LINE User ID ว่าผูกกับผู้ใช้คนไหน
4. ส่งข้อความ/รูปให้ตัวแยกวิเคราะห์ (Parser)
5. บันทึกลงตาราง `transactions` พร้อม `source = line_bot`
6. บอทตอบกลับยืนยัน เช่น "บันทึกรายจ่าย 45 บาท หมวดอาหาร ✅"

## การผูกบัญชี

- เชื่อมบัญชี LINE กับบัญชีในระบบผ่านหน้าตั้งค่า (สแกน QR หรือกดลิงก์เพิ่มเพื่อน)
- เก็บ LINE User ID ไว้ที่ผู้ใช้แต่ละคน (`auth_providers`)
- กรณีคู่รัก: แต่ละคนผูก LINE ของตัวเอง เลือกได้ว่ารายการจะเข้าบัญชีส่วนตัวหรือบัญชีกลาง

## ตัวแยกวิเคราะห์ข้อความ (Parser)

### แบบ Pattern/Format

รูปแบบ: `[ประเภท] [จำนวนเงิน] [หมวดหมู่] [โน้ต]`

ตัวอย่าง:

- `จ่าย 45 อาหาร ข้าวมันไก่`
- `รับ 15000 เงินเดือน`
- `-120 เดินทาง แกร็บ`
- `+500 ฟรีแลนซ์`

### แบบ Natural Language + AI (Phase 4)

ผู้ใช้พิมพ์ภาษาธรรมชาติ เช่น "เมื่อกี้ซื้อกาแฟ 65 บาทที่ starbucks" ระบบส่งให้ AI วิเคราะห์และแยกเป็นจำนวนเงิน หมวดหมู่ ร้านค้า และโน้ตโดยอัตโนมัติ

## คำสั่งที่รองรับในแชท

- บันทึกรายรับ / รายจ่าย
- แก้ไขรายการล่าสุด
- ลบรายการล่าสุด
- สรุปวันนี้ / สรุปเดือนนี้
- ส่งรูปใบเสร็จ → OCR → บันทึกอัตโนมัติ (Phase 4)

## UI เสริมในแชท

- Quick Reply buttons ให้เลือกหมวดหมู่/บัญชีแทนการพิมพ์
- Rich Menu สำหรับเมนูลัด (ดูยอดคงเหลือ, ดูงบประมาณ)

## ค่าใช้จ่าย

| ประเภท | ค่าใช้จ่าย |
|--------|-----------|
| Reply ภายใน 24 ชม. หลังผู้ใช้ทัก | ฟรี |
| Push/Broadcast (เตือนบิลโดยผู้ใช้ไม่ได้ทักก่อน) | มีโควตาฟรีต่อเดือน เกินแล้วคิดตามจำนวนข้อความ |

> ใช้ Push Notification ในแอปเป็นช่องทางหลักสำหรับการแจ้งเตือนเชิงรุก ใช้ LINE เป็นทางเลือกเสริม

> **หมายเหตุ:** LINE Notify ถูกยกเลิกตั้งแต่ 31 มี.ค. 2568 ระบบนี้ใช้ LINE Messaging API ผ่าน LINE Official Account แทน
