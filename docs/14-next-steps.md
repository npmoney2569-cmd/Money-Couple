# สรุปสถานะล่าสุด และ สิ่งที่ต้องทำต่อไป (Next Steps)
อัปเดต: กรกฎาคม 2026

## ✅ สิ่งที่ทำเสร็จแล้วล่าสุด (Recent Updates)
1. **แก้ปัญหา Dashboard โหมดส่วนตัว (Personal Mode)**
   - แก้ไข Dashboard ให้แสดงเฉพาะรายรับ-รายจ่ายของตัวเอง โดยไม่ดึงข้อมูลของคู่รักมาปน
2. **แก้ปัญหารายงาน (Reports) และ ปฏิทิน (Calendar) โหมดส่วนตัว**
   - ใส่ Filter กรองข้อมูลเฉพาะของบัญชีตัวเอง (user_id) ในหน้า Reports และ Calendar เพื่อให้ยอดรวมแสดงผลถูกต้องในโหมดส่วนตัว
3. **แก้ปัญหาระบบลบข้อมูล (Delete)**
   - เปลี่ยนจากการลบถาวร (Hard Delete) เป็นการซ่อนข้อมูล (Soft Delete) โดยใช้วิธีอัปเดตฟิลด์ `deleted_at` 
   - แก้ปัญหาติดสิทธิ์ตารางลูก (RLS cascade issues) ทำให้ตอนนี้สามารถลบรายจ่ายได้แล้ว และระบบ Triggers ทำงานเพื่อคืนยอดเงินเข้าบัญชีได้อย่างสมบูรณ์
4. **AI จัดหมวดหมู่ด้วย Natural Language (Google Gemini)**
   - นำ Gemini API มาเชื่อมต่อกับ LINE Bot ให้ผู้ใช้สามารถพิมพ์บันทึกรายจ่ายด้วยภาษาธรรมชาติ (เช่น "ซื้อข้าว 100") แล้ว AI จะแยกแยะหมวดหมู่และยอดเงินให้เอง
5. **ระบบแจ้งเตือนอัตโนมัติ (Automated Alerts via Vercel Cron)**
   - สร้าง Cron Job ตรวจสอบและยิง Push Notification แจ้งเตือนบิลที่ใกล้ถึงกำหนดเข้า LINE อัตโนมัติทุกเช้า 08:00 น.
6. **ระบบอ่านสลิปและใบเสร็จ (OCR via Gemini AI)**
   - เพิ่มฟีเจอร์ให้ LINE Bot สามารถรับรูปภาพสลิปโอนเงินหรือใบเสร็จ แล้วใช้ AI สกัดยอดเงินและหมวดหมู่บันทึกเข้าบัญชีให้อัตโนมัติ

## 🚀 สิ่งที่ต้องทำต่อไป (Next Steps)

อ้างอิงจาก Roadmap ใน `docs/09-mvp-roadmap.md` และภาพรวมของระบบ สิ่งที่ควรพิจารณาทำเป็นลำดับถัดไปคือ:

### 1. การล็อกอินและการรักษาความปลอดภัย (Phase 3 - Security) **(รอทำทีหลังสุด)**
*   **2FA (Two-Factor Authentication)**: (ถูกเลื่อนไปทำท้ายโปรเจกต์)
*   **PIN Lock / Biometric Lock**: (ถูกเลื่อนไปทำท้ายโปรเจกต์)
*   *หมายเหตุ: Google Login ถูกเตรียมไว้แล้ว หากต้องการเปิดใช้สามารถเข้าไปเซ็ตใน Google Console ได้เลย*

### 2. ฟีเจอร์ระดับสูง (Phase 4 - Advanced Features)
*   **AI วิเคราะห์การเงิน + Forecast**: ให้ AI ช่วยดูแนวโน้มการใช้จ่ายและให้คำแนะนำ
*   **Import Statement ธนาคาร (CSV/Excel)**: อิมพอร์ตข้อมูลย้อนหลังเข้ามาในระบบทีละมากๆ
*   **ระบบ Offline Sync**: รองรับการใช้งานแบบออฟไลน์เมื่อไม่มีเน็ต และซิงค์ขึ้นเซิร์ฟเวอร์ภายหลัง
*   **การห่อแอปเป็น Native (Capacitor/React Native)**: เพื่อนำแอปขึ้น App Store และ Play Store แบบสมบูรณ์

## 🛠️ ข้อเสนอแนะอื่นๆ ด้านเทคนิค (Technical Debts)
*   **การลบข้อมูล (Soft Delete)**: ปัจจุบันเราปรับใช้ Soft Delete กับธุรกรรมรายจ่ายแล้ว ในอนาคตอาจพิจารณาปรับให้รองรับการดึงข้อมูลที่ถูกลบกลับคืนมา (Restore/Undo) ผ่าน UI ถังขยะ (Trash)
*   **Performance / Pagination**: หากข้อมูลมีปริมาณเยอะมาก อาจจะต้องเพิ่มการทำ Infinite Scroll ให้กับหน้าตารางข้อมูลและ Dashboard
