# LINE Bot Integration

## แนวคิด

ผู้ใช้พิมพ์ข้อความหรือส่งรูปใบเสร็จเข้าไปในแชท LINE Official Account ของระบบ แล้วบอทจะแยกวิเคราะห์และบันทึกรายรับ-รายจ่ายให้อัตโนมัติ พร้อมตอบกลับยืนยันในแชท

## Flow การทำงาน

1.  ผู้ใช้พิมพ์ข้อความในแชท LINE เช่น "ข้าวมันไก่ 45" หรือส่งรูปใบเสร็จ
2.  LINE ส่ง event มาที่ Webhook ของระบบ
3.  Backend รับข้อความ ตรวจสอบ LINE User ID ว่าผูกกับผู้ใช้คนไหน
4.  ส่งข้อความ/รูปให้ตัวแยกวิเคราะห์ (Parser)
5.  บันทึกลงตาราง transactions พร้อมระบุ source = line_bot
6.  บอทตอบกลับยืนยันในแชท เช่น "บันทึกรายจ่าย 45 บาท หมวดอาหาร ✅"

## การผูกบัญชี

-   ผู้ใช้เชื่อมบัญชี LINE กับบัญชีในระบบผ่านหน้าตั้งค่า (สแกน QR หรือกดลิงก์เพิ่มเพื่อน)
-   เก็บ LINE User ID ไว้ที่ผู้ใช้แต่ละคน
-   รองรับกรณีคู่รัก: แต่ละคนผูก LINE ของตัวเอง แต่เลือกได้ว่ารายการที่พิมพ์จะเข้าบัญชีส่วนตัวหรือบัญชีกลาง

## ตัวแยกวิเคราะห์ข้อความ (Parser)

**แบบ Pattern/Format**
รูปแบบ: `[ประเภท] [จำนวนเงิน] [หมวดหมู่] [โน้ต]`
ตัวอย่าง: "จ่าย 45 อาหาร ข้าวมันไก่", "รับ 15000 เงินเดือน", "-120 เดินทาง แกร็บ", "+500 ฟรีแลนซ์"

**แบบ Natural Language + AI**
ผู้ใช้พิมพ์ภาษาธรรมชาติ เช่น "เมื่อกี้ซื้อกาแฟ 65 บาทที่ starbucks" ระบบส่งให้ AI วิเคราะห์และแยกเป็นจำนวนเงิน หมวดหมู่ ร้านค้า และโน้ตโดยอัตโนมัติ (ต่อยอดจากฟีเจอร์ AI จัดหมวดหมู่ที่มีอยู่)

## คำสั่งที่รองรับในแชท

-   บันทึกรายรับ / รายจ่าย
-   แก้ไขรายการล่าสุด
-   ลบรายการล่าสุด
-   สรุปวันนี้ / สรุปเดือนนี้
-   ส่งรูปใบเสร็จ → OCR → บันทึกอัตโนมัติ

## UI เสริมในแชท

-   Quick Reply buttons ให้เลือกหมวดหมู่/บัญชีแทนการพิมพ์
-   Rich Menu สำหรับเมนูลัด (ดูยอดคงเหลือ, ดูงบประมาณ)

## ค่าใช้จ่าย

-   การตอบกลับ (Reply) ภายใน 24 ชม. หลังผู้ใช้ทักมา — ไม่มีค่าใช้จ่าย
-   การส่งข้อความเชิงรุกแบบ Push/Broadcast (เช่น เตือนบิลโดยที่ผู้ใช้ไม่ได้ทักก่อน) — มีโควตาฟรีต่อเดือนตามแพ็กเกจของ LINE Official Account จากนั้นมีค่าใช้จ่ายตามจำนวนข้อความ ควรใช้ Push Notification ในแอปเป็นช่องทางหลักสำหรับการแจ้งเตือนเชิงรุก และใช้ LINE เป็นทางเลือกเสริม

> หมายเหตุ: LINE Notify (บริการเดิม) ถูกยกเลิกไปแล้วตั้งแต่ 31 มี.ค. 2568 ระบบนี้ใช้ LINE Messaging API ผ่าน LINE Official Account แทน

# Platform Support

## แนวทาง

เป็น Web App เดียว รองรับทุกอุปกรณ์ผ่าน Responsive Design + PWA (Progressive Web App) ไม่ต้องพัฒนาแอปแยกสำหรับแต่ละแพลตฟอร์ม

## อุปกรณ์ที่รองรับ

-   PC (Windows / Mac) — ผ่านเบราว์เซอร์ปกติ
-   Android (มือถือ/แท็บเล็ต) — เบราว์เซอร์ หรือติดตั้งเป็น PWA
-   iOS (iPhone/iPad) — Safari หรือติดตั้งเป็น PWA ("เพิ่มลงหน้าจอหลัก")

## Responsive Design

-   จอมือถือ: Bottom Navigation, ฟอร์มเต็มจอ, กราฟแบบย่อ
-   จอแท็บเล็ต/iPad: Layout 2 คอลัมน์ (รายการ + รายละเอียด)
-   จอ PC: Sidebar ถาวร, Dashboard แสดงข้อมูล/กราฟเต็มรูปแบบ

## PWA Features

-   Web App Manifest (ไอคอน, ชื่อแอป, ธีมสี)
-   Service Worker — รองรับ Offline Sync และ Push Notification ผ่านเบราว์เซอร์
-   Add to Home Screen — ใช้งานเหมือนแอปจริง ไม่มีแถบ URL

## ความปลอดภัยระดับอุปกรณ์

-   Biometric Lock (Face ID / Touch ID / Fingerprint) ผ่าน WebAuthn API

## แนวทางขยายในอนาคต

-   Wrap เป็นแอป Native เต็มรูปแบบด้วย Capacitor หรือ React Native WebView เพื่อขึ้น App Store / Play Store โดยใช้โค้ดเว็บเดิมเป็นฐาน

# ความปลอดภัย & การยืนยันตัวตน

## Login / Register

รองรับ Login แบบผสม เลือกวิธีใดวิธีหนึ่งเพื่อสมัคร/เข้าระบบ:

-   Email + Password
-   Google Login
-   Apple Login
-   LINE Login

## ผูกช่องทางเพิ่มเติมภายหลัง (Account Linking)

-   ผู้ใช้สมัครด้วยวิธีใดวิธีหนึ่งก่อน แล้วเข้าไปที่หน้าตั้งค่า → "บัญชีที่เชื่อมต่อ" เพื่อผูกช่องทางอื่นเพิ่มได้ภายหลัง เช่น
    -   สมัครด้วย Email → ผูก LINE เพิ่มทีหลังเพื่อเปิดใช้ LINE Bot
    -   สมัครด้วย LINE Login → ผูก Email เพิ่มทีหลังเพื่อกู้คืนบัญชีกรณีเข้า LINE ไม่ได้
-   ทุกช่องทางที่ผูกแล้วใช้ Login เข้าบัญชีเดียวกันได้ (อ้างอิงจาก user_id เดียวกัน)
-   ป้องกันการผูกอีเมล/LINE ที่มีคนอื่นใช้อยู่แล้วซ้ำ (unique constraint)
-   ยกเลิกการผูกช่องทางได้ ยกเว้นกรณีเหลือช่องทางเข้าระบบเพียงช่องทางเดียว (ต้องมีอย่างน้อย 1 ช่องทางเสมอ)
